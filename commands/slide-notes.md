Generate speaker notes for an existing presentation outline.

The user will provide an existing slide deck outline or slide titles. Use the following input: $ARGUMENTS

If no outline is provided in the arguments, ask the user to paste their slide outline or provide a file path.

## Instructions

For each slide in the provided outline, generate speaker notes that cover:

1. **What to say:** Conversational talking points the presenter should deliver. Not a word-for-word script, but structured enough to keep the presenter on track. 3-5 sentences per slide.

2. **Transition:** A natural bridge sentence leading into the next slide. This should feel smooth, not abrupt.

3. **Audience cues:** Note where to pause for effect, ask the audience a question, reference a demo, or check for understanding.

4. **Timing guidance:** Suggest how long to spend on this slide.

## Guidelines

- Write in a natural speaking voice. These notes are meant to be glanced at, not read word for word.
- Avoid repeating what is already on the slide. The notes should ADD context, stories, or explanations.
- If a slide has technical content, include a brief analogy or plain-language explanation the presenter can use to make it accessible.
- Flag any slides that feel thin (not enough to talk about) or overloaded (too much for one slide).
- If the outline is missing a title slide or closing slide, note that and suggest what to add.
